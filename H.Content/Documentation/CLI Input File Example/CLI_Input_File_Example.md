# Example Input Tables For Command Line Interface

This document provides a detailed reference for each column (field) in the user-created CSV input files that are processed by the Holos CLI. Please read [here](https://github.com/holos-aafc/Holos/blob/main/H.Content/Documentation/User%20Guide/User%20Guide.md#chapter-10---command-line-interface) for more information on the Holos CLI.

## How to Use This Document

Each section below corresponds to a **component type** (Field, Dairy, Beef, Sheep, or Farm Setting). Within each section, every column header in the CSV input file is documented with the following information:

- **Example value**: A sample value you might enter in this column.
- **Type value**: The data type expected (e.g., Double, Int, Enum, Bool, GUID, String).
- **Does user have to provide value**: Whether the user must enter a value, or whether the application will assign a default.
- **Holos has a default value**: Whether Holos will auto-populate this field if left blank or set to "N/A".
- **Valid values / Valid range of values**: The accepted inputs for this column.
- **Note**: Additional context on how this column is used in calculations.
- **Source**: Links to the relevant source code files or reference tables.

## Important Notes Before You Begin

- **File format**: All input files must be saved as **.csv** (comma-separated values). Microsoft Excel is recommended for editing these files. Other spreadsheet software (e.g., OpenOffice) may introduce formatting errors.
- **CSV delimiters**: The delimiter used in your CSV files depends on your system's language setting. For **English** locales (en-CA, en-US), use a **comma (,)** as the delimiter and a **period (.)** as the decimal separator. For **French (Canada)** locale (fr-CA), use a **semicolon (;)** as the delimiter and a **comma (,)** as the decimal separator.
- **Column ordering**: The columns in your CSV file must appear in the same order as they are listed in this document for each component type. The CLI will read columns by their position. The column headers in the CSV file must exactly match the header names shown in the template files generated by the CLI.
- **Not applicable values**: If a field is not relevant to your scenario, you can enter **"N/A"** (not case sensitive). The CLI will treat this as a blank/not-applicable entry.
- **Imperial vs. Metric units**: When you select **Imperial** units at the CLI startup prompt (or via the `-u` command line flag), certain columns will be automatically converted from Imperial to Metric internally by the CLI before calculations are performed. The specific columns affected are noted in each field's documentation. When using **Metric** units, no conversion is applied and your values are used as entered.
- **Deprecated columns**: Some columns are marked as **"Deprecated. Do not use."** in their notes. These columns exist for backward compatibility with older input files but are no longer used in calculations. You can enter any value (or "N/A") for these columns. They will be removed in a future version of Holos.
- **Template files**: When the CLI creates a new farm folder, it generates template CSV files with all the correct column headers pre-populated. These template files have filenames ending in "_Example-en-CA.csv" or "_Example-fr-CA.csv". Use these templates as a starting point for your own input files. Copy the template, rename it (removing "Example" from the filename), and fill in your data.
- **One row per year**: For Field input files, each row in the CSV represents data for a single crop year. If you have data spanning multiple years, you will have multiple rows in the same file.

# Field

The Field input file contains data about crop production, tillage, fertilizer, and other land management activities. Each row represents a single crop year for a field component. Fields and Shelterbelts are processed first by the CLI because other components (e.g., Swine with pasture housing) may reference field data.

# Dairy

The Dairy input file contains data for dairy cattle operations. Each row represents a single **management period** for an **animal group** within a dairy component. A dairy component can contain multiple animal groups (e.g., heifers, lactating cows, calves), and each animal group can have multiple management periods throughout the year with different housing, diet, and management practices.

**Key concepts for animal input files (applies to Dairy, Beef, Sheep):**

- **Component**: A broad category of animals on your farm (e.g., "Dairy Heifers", "Backgrounding Beef"). Each component is represented by a separate CSV input file.
- **Animal Group**: A group of animals within a component that share the same characteristics (e.g., "Heifers group #1"). Identified by the GroupName column.
- **Management Period**: A time span during which a group of animals is managed consistently (same housing, diet, etc.). Defined by ManagementPeriodStartDate and ManagementPeriodDays.
- **Group Pairing**: Used to link parent and offspring groups (e.g., lactating cows and their calves). Groups that are paired share the same GroupPairingNumber value.

# Farm Setting

The Farm Setting file (with a .settings extension) contains global configuration parameters for a farm. These settings control how Holos performs calculations across all components on the farm, including which carbon model to use, climate data sources, yield assignment methods, and various soil carbon modelling parameters. Each farm folder must contain at least one .settings file. You can have multiple .settings files per farm to run different scenarios with different configurations.
